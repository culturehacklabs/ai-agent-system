## How to Use This KB

When a narrative practitioner asks:
- *"What do I need to document?"* → go to the relevant phase section below
- *"Is my documentation complete?"* → check Required fields for that phase
- *"I'm missing X — does it matter?"* → check whether it is Required or Optional
- *"Where do I build this?"* → navigate them to the relevant phase agent

Always distinguish **Required** (cannot be skipped — needed for evaluation) from **Optional** (enhances documentation but not blocking).

## Phase 1 — Ask: Point of View

**Required:**
- Full POV statement: *"We are [identity] / we see that [problem] / Therefore we want [vision] / To do this, we must [theory of change]"*
- Collective identity (not individuals)
- Narrative observation (the cultural construct or frame being challenged — not just a material condition)
- Vision (what shifts if the narrative changes)
- Theory of change (how narrative shift connects to systems transformation)
- Short POV (1–3 sentences — this is the handoff to Agent 02a)
- Short-term goals (6 months–2 years)

**Optional:**
- Long-term goals (5–10 years)
- Systems change contribution statement
- Connection to a Rhizome Fellowship Pathway
- Links to external documents

**Critical insight:** The POV names a *narrative problem*, not just a social condition. "Deforestation is increasing" is a material condition. "Nature is framed as a resource available for extraction" is a narrative problem. Agent 01 builds this.

## Phase 2 — Listen: Listening Model

**Required:**
- Restated Short POV
- Listening objectives (2–4 specific questions this research must answer)
- For each Small Listening action (minimum 1, recommended 1–3):
  - What (specific themes, keywords, frames to track)
  - Who (specific communities, actors, demographics)
  - When (date range + reasoning)
  - Where (platforms, geography, physical/digital spaces)
  - How (research method + documentation format + storage system)
- Big Listening inquiry: specific research question + What/Who/When/Where parameters

**Optional:**
- Connection to Rhizome Fellowship Pathway
- Connection to other practitioners' research questions

**Critical insight:** Documentation discipline here is essential. Interviews must be recorded and transcribed. Media sources tracked with links. Observations in consistent formats. The data collected here is the baseline — without it, there is nothing to compare against after the hack. Agent 02a designs the model; Agent 02b executes the research.

## Phase 3 — Understand: Narrative Communities

**Required:**
- 3–5 narrative communities identified and described
- For each community:
  - Name + 1–2 sentence description
  - Core narrative (how they explain the issue, its causes, solutions)
  - Key actors (organisations, individuals, publications)
  - Language patterns (vocabulary, metaphors, hashtags)
  - Frame analysis (inside / outside the frame / how common sense is constructed)
  - Platform presence and geographic reach
  - Community type: Hegemonic / Alternative / Catalytic (with reasoning)
  - ANP assessment (Attention / Network / Power — High/Medium/Low with evidence)
- Community Map (Justice/Ontology axis positions for each community, with reasoning)
- Window of Discourse assessment (Mainstream / Emerging / Radical positions)
- Narrative Space Summary (dominant patterns, key tensions, strategic opportunities, baseline state)

**Optional:**
- Sub-community splits if a community proves more complex than initially mapped
- Bridge actor identification
- Detailed network topology

**Critical insight:** These community profiles are the "before" picture. After the hack, the practitioner will compare against them to show what shifted. Agent 03 builds these.

## Phase 4 — Recode: Narrative Strategy

**Required:**
- Narrative objectives:
  - Target communities (2–3, with reasoning and desired direction of movement)
  - Frame transformation plan (frames to challenge / strengthen / introduce)
  - Window of Discourse current position + desired movement
- Decode table: full 8-element analysis of the dominant narrative (or own message)
  - Elements: Verbs, Subjects & Objects, Evoked Images, Hidden Assumptions, Logic, Ideology, Metaphor, Framing
- Recode table: full 8-element construction of the alternative narrative
- Exact reframed message (specific language, every word a strategic choice)
- Target audience definition

**Optional:**
- Multiple message iterations documented
- Testing with target community members + refinements made

**Critical insight:** The message is not a slogan. It is the structure of meaning the hack builds on. Agent 04 builds this.

## Phase 5 (05a) — Hack: Hacking Tree + Logistics

**Required:**
- Full Hacking Tree:
  - **Media:** format, platform, audience fit reasoning
  - **Meme:** symbol chosen, association remix (current → new meaning), synthesis (exact hashtag/slogan/image/gesture)
  - **Tone:** current emotional mood of narrative space, feelings to generate, emotional journey, creative stimuli
  - **Hack — Space:** where (type + specific location/platform + symbolic significance)
  - **Hack — Scale:** Micro/Meso/Macro/Mega + realistic reach + point of intervention
  - **Hack — Moment:** timing (planned / event-responsive / cyclical) + wave assessment
- Logistics:
  - Timeline (Development → Production → Launch → Sustained → Evaluation phases)
  - Team roles
  - In-house capacities + external needs
  - Budget (minimum viable + optimal)
  - Documentation plan (before / during / after)
  - Monitoring plan (what to track + how often + by whom)

**Optional:**
- Distribution partner list
- Contingency plan if moment shifts
- Secondary audience mapping

**Critical insight:** The monitoring plan set here becomes the data collection infrastructure for Agent 05b. Without it, evaluation will be anecdotal. Agent 05a builds this.

## Phase 5 (05b) — Evaluate: Impact + Iteration (nested in Hack)

**Required:**
- Confirmation that baseline data is accessible (from Listen/Understand phases)
- Level 1 — Audience Impact:
  - Reach (quantitative: how far did it travel; vs. baseline)
  - Response (quantitative + qualitative engagement)
  - Adoption (early signals of new frames/language/behaviours in target audience)
- Level 2 — Discourse Impact:
  - Attention shifts (vs. ANP baseline from Agent 03)
  - Network shifts (new connections, bridge actors)
  - Power shifts (legitimacy/reach changes)
  - Window of Discourse movement (vs. baseline position)
- Level 3 — Systems Change:
  - Early signals of broader shift
  - Connection to larger movement ecosystem
  - Contribution to longer-term tipping point conditions
- Case Study (full arc: POV → Listen → Understand → Strategy → Hack → Observed → Learned)
- Iteration Plan (what the next cycle needs: updated listening, revised strategy, different timing/audience)

**Optional:**
- Advanced Case Study format (full field-by-field documentation for movement knowledge-sharing)
- Quantitative evaluation tools (surveys, A/B testing, sentiment analysis)

**Critical insight:** Evaluation is not a verdict — it is the feedback loop that makes each cycle more effective. Agent 05b runs this.

## Handling Documentation Gaps

When a practitioner has incomplete documentation:

1. **Identify the gap** — which phase, which field
2. **Assess criticality** — is it Required or Optional?
3. **Check if retrievable** — can they still gather this information?
4. **Guide completion** — give them the specific question to answer
5. **Proceed if possible** — if the gap is in Optional fields, document the limitation and move forward

For critical gaps in early phases (POV not clear, no baseline data), name the impact plainly: without a clear POV, the hack lacks direction; without a baseline, impact cannot be demonstrated. Then navigate them back to the relevant agent to fill the gap before proceeding.
