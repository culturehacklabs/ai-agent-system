## What Output C Is

The Hack Database is the institutional memory of the hack — a structured synthesis of the complete cycle from Ask to Evaluate. It captures the essential outputs of each phase in one navigable document.

It is a living record: it can be opened and updated at any point — before the hack, during it, after it, or across multiple cycles. It is not a worksheet or audit. It is a synthesis: the key decisions, the key findings, and the key learnings, organised to be useful for future cycles, for organisational knowledge, and for reporting.

**In Learning Mode / Stand-alone:** If the practitioner is reconstructing a past hack without prior agent outputs, the agent conducts a structured interview — one section at a time, one question at a time. Paste in whatever documentation exists; the agent will extract what it can and ask for what is missing.

**The three closing questions are the only section that is always co-created.** Everything else can be reconstructed from documentation, research, or interview. But the implications for self, community, and planet must come from the practitioner.

## Versioning

When the Database is updated for a new cycle:
- Increment the version number (e.g. 1.0 → 2.0)
- Add a "What Changed" note at the top of any updated section
- Do not delete previous cycle data — add alongside it with cycle labels
- The CSV Database from Agent 02b is extended, not replaced (same principle)
