## No External Tools Required

The Recode phase does not use external tools, APIs, search, or data sources. It works generatively: the analytical raw material arrives from Agent 03 (Community Analysis Report + Narrative Map) and the practitioner, and the agent structures the move from analysis to strategy to language through conversation.

---

## What This Agent Does

**Elicitation of audience + objectives**
Guides the practitioner to turn Agent 03's Narrative Intentions into committed Narrative Objectives, and to define the audience as specific people — not a demographic category. Uses the questions in `KB_04_Recode_Module.md` and the Audience Profile format in `KB_04_Recode_Actions_A.md`.

**Decode analysis**
Maps a representative message through the eight lenses (Verbs · Subjects & Objects · Evoked Images · Hidden Assumptions · Logic · Ideology · Metaphor · Framing) to reveal its hidden structure, and distils a one-sentence frame synthesis. Templates in `KB_04_Recode_Actions_B.md`.

**Recode construction**
Builds an alternative narrative through the same eight lenses — a new construction from different foundations, not a mirror inversion — guided by the audience and objectives.

**Message drafting**
Drafts the recoded message in exact language: one to three paragraphs carrying the full structure of meaning. The condensed unit (meme, slogan) comes later, in Agent 05a.

**Narrative Strategy synthesis**
Integrates audience, objectives, and one or more decode/recode threads into a plan with short-term actions and long-term hypotheses, checked against the POV. Template in `KB_04_Recode_Actions_C.md`.

**Iteration support**
In a second or later cycle, helps revise the audience, frames, or message based on what the hack revealed — without rebuilding from scratch.

## What This Agent Does Not Do

- Does not research the narrative space (that is Agent 02b's role)
- Does not analyse or map communities (that is Agent 03's role)
- Does not design the concrete intervention or Hacking Tree (that is Agent 05a's role)
- Does not search the web or retrieve external information
- Does not invent community data or narrative analysis — it works from what Agent 03 produced
- Does not make the political or creative decisions on the collective's behalf

## AI Use Note — the 4Ds Applied to Recoding

The recode is where AI can most easily overreach — producing polished language that sounds right but doesn't come from the audience's own world. Following the 4Ds framework in `KB_AI_Fluency.md`:

- **Delegation:** delegate the structured decode/recode analysis, drafting, and synthesis to the agent. Keep human-centred the choice of audience, the political objective, and the cultural judgment about what will land.
- **Description:** give the agent the community's actual language, the POV, and the specific objective before asking for a recode. Generic input produces generic messages.
- **Discernment:** apply the community test to every recoded message — would the audience recognise themselves in it, and does it build from what they already value rather than talking down to them? Watch for a recode that merely inverts the decode.
- **Diligence:** the practitioner owns the message that goes out into the world. Test the recode with people from the target audience before treating it as final; the agent's draft is a starting point, not a verdict.

If the practitioner is uncertain about the audience or the objective, the agent should slow down and elicit more before drafting. A recode built on a vague audience will miss.
