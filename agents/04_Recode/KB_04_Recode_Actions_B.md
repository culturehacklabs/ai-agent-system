## Step 1 — Select the Representative Message

Before any analysis begins, the practitioner must identify a specific paragraph-length message to decode.

**What makes a good representative message:**
- It is a real text — not a summary or paraphrase of a narrative
- It is paragraph-length: enough to contain logic, framing, and ideological structure
- It exemplifies the dominant narrative the hack is working against
- It is already visible in the 02b/03 outputs: a media excerpt, a campaign statement, a public speech, a community quote

**Where to look:**
- The CSV Database from Agent 02b (content samples column)
- The ANP Source Library entries flagged for the Hegemonic community
- Language analysis notes from Agent 03 on dominant frames

**If the practitioner is decoding their own existing message:**
- Identify the specific text (a communication, a campaign message, a fundraising ask)
- Note what prompted the review — what do they suspect it might be doing unintentionally?

**If no message has been identified:**
The agent should help the practitioner:
1. Return to the 02b/03 outputs and look for the most-cited or most-reach framing from the Hegemonic community
2. If none is there, ask: what is the single most common thing the target audience *hears* about this topic? Draft a representative paragraph together.

## Multiple Decode/Recode Cycles

If the strategy requires engaging more than one audience with different messages, produce one Decode + Recode pair per audience thread. Each pair feeds separately into the Narrative Strategy (Actions C).

Label each thread clearly:
- **Thread 1:** Audience [name] — Decode [source] → Recode [message]
- **Thread 2:** Audience [name] — Decode [source] → Recode [message]

The Strategy integrates all threads into a coherent action plan.
